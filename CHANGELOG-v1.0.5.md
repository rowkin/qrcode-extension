# v1.0.5 更新日志

## 🎨 新功能：自定义二维码颜色

### 功能特性

1. **颜色自定义**
   - 支持自定义前景色（二维码图案颜色）
   - 支持自定义背景色（二维码背景颜色）
   - 使用 HTML5 color input 选择器
   - 支持手动输入颜色值（十六进制格式）

2. **预设颜色主题**
   - 经典黑白：黑色前景 + 白色背景
   - 蓝色主题：蓝色前景 + 白色背景
   - 绿色主题：绿色前景 + 白色背景
   - 红色主题：红色前景 + 白色背景
   - 紫色主题：紫色前景 + 白色背景

3. **实时预览**
   - 选择颜色后立即重新生成二维码
   - 使用防抖优化，避免频繁生成
   - 平滑的动画效果

4. **持久化存储**
   - 使用 chrome.storage.local 保存用户选择
   - 下次打开时自动恢复上次设置
   - 支持重置为默认颜色

### UI 设计

- **折叠式设计**：颜色选择器默认折叠，点击展开
- **响应式布局**：适配移动端和桌面端
- **直观操作**：颜色选择器 + 文本输入 + 预设按钮

### 技术实现

- 添加 `storage` 权限到 manifest.json
- 使用 `chrome.storage.local` API 存储颜色设置
- 更新 `generateQRCode` 函数支持自定义颜色
- 添加 `setupColorCustomizer` 函数处理颜色选择
- 使用防抖（debounce）优化性能

### 国际化支持

- 中文：完整的中文界面
- 英文：英文界面支持
- 日文：日文界面支持

### 文件变更

- `manifest.json`：版本更新到 1.0.5，添加 storage 权限
- `content-script.js`：添加颜色选择器 UI 和功能
- `_locales/*/messages.json`：添加颜色相关文本
- `README.md` / `README-zh.md`：更新文档

### 使用说明

1. 生成二维码后，点击"自定义颜色"按钮
2. 展开颜色选择面板
3. 选择前景色和背景色：
   - 使用颜色选择器
   - 或手动输入颜色值（如 #1a73e8）
   - 或点击预设颜色按钮
4. 颜色会实时应用到二维码
5. 设置会自动保存，下次打开时恢复

### 注意事项

- 确保前景色和背景色有足够对比度，以保证二维码可扫描性
- 建议使用深色前景 + 浅色背景的组合
- 颜色设置会保存到浏览器本地存储

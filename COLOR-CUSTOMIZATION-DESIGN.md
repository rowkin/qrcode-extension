# 自定义二维码颜色功能设计文档

## 🎨 交互方案设计

### 设计目标
- 简单易用：用户能快速选择颜色
- 实时预览：选择颜色后立即看到效果
- 持久保存：用户选择的颜色会被保存
- 美观实用：不影响现有 UI 的简洁性

### UI 设计方案

#### 方案一：折叠式颜色选择器（推荐）
- 在二维码面板底部添加一个"自定义颜色"按钮
- 点击后展开颜色选择区域
- 包含前景色和背景色两个颜色选择器
- 提供预设颜色快速选择
- 支持颜色输入框手动输入

#### 布局设计
```
┌─────────────────────────┐
│      [关闭按钮]          │
│                         │
│   [二维码显示区域]       │
│                         │
│   [URL 显示和复制]       │
│                         │
│   [自定义颜色 ▼]         │ ← 可折叠按钮
│   ┌─────────────────┐  │
│   │ 前景色: [色块]   │  │ ← 展开区域
│   │ 背景色: [色块]   │  │
│   │ [预设颜色]       │  │
│   │ [重置] [应用]    │  │
│   └─────────────────┘  │
└─────────────────────────┘
```

### 功能特性

1. **颜色选择器**
   - 前景色（二维码图案颜色）
   - 背景色（二维码背景颜色）
   - 使用 HTML5 color input
   - 提供预设颜色快速选择

2. **实时预览**
   - 选择颜色后立即重新生成二维码
   - 无需点击"应用"按钮

3. **预设颜色**
   - 经典黑白
   - 蓝色主题
   - 绿色主题
   - 红色主题
   - 紫色主题

4. **持久化存储**
   - 使用 chrome.storage.local 保存用户选择
   - 下次打开时自动恢复

5. **重置功能**
   - 一键恢复默认颜色（黑白）

## 📋 实现步骤

### 步骤 1：添加颜色存储功能
- 使用 chrome.storage.local 存储颜色设置
- 默认值：前景色 #000000，背景色 #ffffff

### 步骤 2：更新面板 UI
- 在面板中添加颜色选择器区域
- 添加折叠/展开功能

### 步骤 3：更新生成函数
- 修改 generateQRCode 函数支持自定义颜色
- 从存储中读取颜色设置

### 步骤 4：添加实时预览
- 颜色改变时重新生成二维码
- 优化性能，避免频繁生成

### 步骤 5：更新国际化
- 添加颜色相关的文本翻译

### 步骤 6：测试和优化
- 测试各种颜色组合
- 确保二维码可扫描性
- 优化用户体验

## 🎯 技术实现要点

1. **颜色对比度**
   - 确保前景色和背景色有足够对比度
   - 提供对比度检查提示

2. **性能优化**
   - 使用防抖（debounce）避免频繁重新生成
   - 缓存颜色设置

3. **兼容性**
   - 确保所有浏览器支持 color input
   - 提供降级方案

4. **用户体验**
   - 平滑的展开/收起动画
   - 清晰的视觉反馈
   - 直观的操作流程
